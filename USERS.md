# OpenSalesTax — Users / Deployers

This file is a public registry of organizations and projects using
OpenSalesTax in production. Listing here is **voluntary and free**.

Per [`specs/constitution.md`][const] §2 / §14, this registry is
OpenSalesTax's primary mechanism for encouraging contribute-back
behavior from SaaS deployers and commercial integrators in lieu of
AGPL-style legal enforcement. **Recognition is currency.**

[const]: specs/constitution.md

## Self-hosters and integrators

_None yet — OpenSalesTax is pre-1.0._

## How to get listed

Open a PR adding your entry to the relevant section below. We accept
entries from:

- **Self-hosting organizations** running OpenSalesTax in production
  for internal use.
- **SaaS providers** whose service is built on or includes
  OpenSalesTax.
- **Commercial integrators** (accounting software, e-commerce
  platforms, ERPs) embedding OpenSalesTax.
- **Open-source projects** that depend on OpenSalesTax (e.g., the
  eventual SC Books integration).

### Listing format

```markdown
## Self-hosters and integrators

### <Organization or project name>
- **Use case:** <one sentence on what you do with OpenSalesTax>
- **Production since:** <YYYY-MM>
- **Website:** <https://your-site.example.com>
- **Contributed back:** <links to merged PRs, or "not yet">
```

### Why we ask "contributed back"

Per the project's [contribute-back norms][const-2-norms], we publicly
note whether listed deployers have upstreamed improvements. This is
explicitly NOT a prerequisite for being listed — but it's how the
project tracks the health of the recognition-over-enforcement
contributor model.

If you're using OpenSalesTax and have ideas, fixes, or per-state
expertise to share, please contribute. The state maintainer pattern
(see [`MAINTAINERS.md`](MAINTAINERS.md)) is the most welcomed
contribution channel.

[const-2-norms]: specs/constitution.md

## Removal

Want to be unlisted? Open a PR or issue and we'll remove your entry
no questions asked.
